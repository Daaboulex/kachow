# Verify AI Context Sync

Comprehensive sync check across the full context chain: project instructions, commands/skills, rules, memory, hooks, and global config.

## Architecture (know this before checking)

The context system has 3 layers:
1. **Per-repo**: `repos/<name>/CLAUDE.md` + `GEMINI.md` (generated by `.ai-context/repo-standard/sync.sh`)
2. **Project**: `.ai-context/CLAUDE.md` + `.gemini/` (symlinked), `.ai-context/memory/` (portable), `.claude/rules/`
3. **Global**: `~/.claude/CLAUDE.md`, `~/.claude/hooks/`, `~/.gemini/hooks/`

Sync is maintained by hooks:
- `sync-gemini-md.js` (Claude PostToolUse) — CLAUDE.md → GEMINI.md (line-by-line, protects sync-table rows)
- `sync-claude-md.js` (Gemini AfterTool) — GEMINI.md → CLAUDE.md (same protection)
- `sync-gemini-skills.js` / `sync-claude-skills.js` — bidirectional command/skill/rule sync

**Critical rule:** Sync hooks MUST be PostToolUse (Claude) / AfterTool (Gemini). Never Pre/Before.

Global hook repos: `<owner>/<repo>`, `<owner>/<repo>` (private)

## Process

### 1. Compare commands/skills
- Claude: `.claude/commands/*.md`
- Gemini: `.gemini/skills/*/SKILL.md`
- Report missing in either direction

### 2. Compare rules
- `.claude/rules/*.md` vs `.gemini/rules/*.md`
- For matching names, diff content (normalize agent names first)

### 3. Compare CLAUDE.md ↔ GEMINI.md (the critical check)
Normalize both files by replacing agent-specific terms with placeholders:
```bash
sed -e 's/Claude/{{AGENT}}/g' -e 's/claude/{{agent}}/g' -e 's/CLAUDE/{{AGENT_UPPER}}/g' \
    -e 's/Gemini/{{AGENT}}/g' -e 's/gemini/{{agent}}/g' -e 's/GEMINI/{{AGENT_UPPER}}/g' \
    -e 's/\.claude\//\.{{dir}}\//g' -e 's/\.gemini\//\.{{dir}}\//g' \
    -e 's/commands/{{cmds}}/g' -e 's/skills/{{cmds}}/g' \
    -e 's/Slash Commands/{{section}}/g' -e 's/Agent Skills/{{section}}/g'
```
Diff the normalized versions. Any remaining differences are REAL drift.

**Known safe differences** (not drift): `CLAUDE.md` vs `GEMINI.md` in the title line.

**Known corruption pattern**: If sync-table rows (containing `→` and `|`) show ``.gemini/skills/` → `.gemini/skills/`` instead of ``.claude/commands/` → `.gemini/skills/``, the sync hook has the old bug. Fix: update `~/.claude/hooks/sync-gemini-md.js` from `[claude-repo]` repo.

### 4. Check portable memory
- Verify `.ai-context/memory/MEMORY.md` exists
- Verify the symlink `~/.claude/projects/<sanitized>/memory/` → `.ai-context/memory/` is intact
- List any memory files not in MEMORY.md index

### 5. Check shared hook file sync
Shared hooks MUST be identical between agents (Claude is source of truth):
```bash
# Check shared hooks are in sync
for f in dream-auto.js dream-session-counter.js; do
  diff <(sed '1d' ~/.claude/hooks/$f) <(sed '1d' ~/.gemini/hooks/$f) > /dev/null 2>&1 \
    && echo "$f: OK" || echo "$f: DRIFT"
done
```
Also check both global repos have no uncommitted changes:
```bash
git -C ~/.claude status --porcelain | head -5
git -C ~/.gemini status --porcelain | head -5
```
The `auto-push-global.js` hook auto-commits+pushes on session end (30-min cooldown) and syncs shared hooks from Claude→Gemini. If drift is found, run the auto-push manually: `node ~/.claude/hooks/auto-push-global.js < /dev/null`

### 6. Check settings.json platform parity

Both `~/.claude/settings.json` and `~/.gemini/settings.json` should have equivalent hooks. Cross-check:

```bash
echo "=== Claude hooks ===" && jq -r '.hooks | to_entries[] | "\(.key): \(.value | length) groups"' ~/.claude/settings.json
echo "=== Gemini hooks ===" && jq -r '.hooks | to_entries[] | "\(.key): \(.value | length) groups"' ~/.gemini/settings.json
```

**Verify event name mapping:**
- Claude `Stop` should correspond to Gemini `SessionEnd`
- Claude `PreToolUse` → Gemini `BeforeTool`
- Claude `PostToolUse` → Gemini `AfterTool`
- Claude `SubagentStart` → Gemini `BeforeAgent`
- Claude `SubagentStop` → Gemini `AfterAgent`
- Claude `PreCompact` → Gemini `PreCompress`
- Claude-only events (no Gemini equivalent): `InstructionsLoaded`, `StopFailure`, `TaskCompleted`

**Verify sync hooks are AfterTool/PostToolUse (NEVER Before/Pre):**
```bash
jq -r '.hooks.BeforeTool[]?.hooks[]?.command' ~/.gemini/settings.json 2>/dev/null | grep -i sync && echo "BUG: sync hook under BeforeTool"
jq -r '.hooks.PreToolUse[]?.hooks[]?.command' ~/.claude/settings.json 2>/dev/null | grep -i sync && echo "BUG: sync hook under PreToolUse"
```

**Verify timeout units:**
- Claude: seconds (single digits like `5`, `20`)
- Gemini: milliseconds (thousands like `5000`, `20000`)

```bash
# Check for suspiciously low Gemini timeouts (likely seconds instead of ms)
jq -r '[.hooks[][] | .hooks[]? | .timeout // empty] | map(select(. < 100))' ~/.gemini/settings.json
```

**Verify translation pattern count parity:**
```bash
echo "Forward (Claude→Gemini): $(grep -c '\.replace(' ~/.claude/hooks/sync-gemini-md.js) patterns"
echo "Reverse (Gemini→Claude): $(grep -c '\.replace(' ~/.gemini/hooks/sync-claude-md.js) patterns"
```

### 7. Check per-repo CLAUDE.md freshness
- For each repo in `repos/`, verify CLAUDE.md exists
- Check if `.ai-context/repo-standard/repos.json` `standardVersion` matches what's deployed
- Report repos that may need `bash .ai-context/repo-standard/sync.sh` re-run

### 7. Report

```
## Context Sync Report

Commands/Skills: [N] matched, [M] Claude-only, [K] Gemini-only
Rules: [N] matched, [M] Claude-only, [K] Gemini-only
Instructions: [OK | DRIFT — details]
Memory: [N] files, symlink [OK|BROKEN], index [OK|STALE]
Global hooks: [OK | DIVERGED — details]
Per-repo standard: [N]/[M] repos up to date

### Drift Details (if any)
- [file]: [issue description]

### Recommended Fixes
- [action to take]
```

### 8. Offer to fix
For each issue found:
- Missing command/skill → generate from the other agent's version
- CLAUDE.md ↔ GEMINI.md drift → re-trigger the sync hook by touching CLAUDE.md
- Broken memory symlink → re-run `ensure-portable-memory.js`
- Global hook divergence → push local to repo, or pull repo to local
- Stale per-repo CLAUDE.md → run `bash .ai-context/repo-standard/sync.sh`

### 9. Self-improvement
After running, check: did this skill catch everything? If a new sync mechanism was added (new hook, new file type, new layer) that this skill doesn't check, flag it:
> "verify-sync does not check [X] — consider updating the skill"

## Red Flags

If you catch yourself thinking any of these, STOP — you're rationalizing:

| Thought | Reality |
|---------|---------|
| "I only edited one file, sync is fine" | One edit in CLAUDE.md cascades to GEMINI.md, context-index, and skill-lineage. |
| "The sync hooks will catch it" | Sync hooks have bugs. verify-sync EXISTS because they're not 100% reliable. |
| "I'll verify next session" | Drift compounds. By next session you'll have forgotten what changed. |
| "It's just a comment change, doesn't matter" | Comment changes in SKILL.md affect description parsing and model routing. |
